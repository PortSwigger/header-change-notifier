Header Change Notifier is a professional Burp Suite extension designed to monitor and detect changes in HTTP response headers. As headers change between requests made to the same URL, an alert is triggered. This extension is an excellent tool for security researchers, penetration testers, and bug bounty hunters who actively seek to identify security misconfigurations via header changes.

Features

  • Real-time Header Monitoring: The extension detects automatic changes in HTTP response headers.
  • Security-Focused: The tool is already pre-configured to keep track of critical security headers.
  • Risk Assessment: It categorizes changes by risk level from Critical/High/Medium/Low.
  • Professional UI: It is embedded with a clean, table-based interface that integrates perfectly into Burp Suite.
  • Customizable Tracking: Users can add or remove headers to be monitored in accordance with their needs.
  • Export Functionality: The module can export all detected changes for reporting in CSV format.
  • Burp Suite Integration: The extension creates alerts in Burps issue tracker for any high-risk changes.
  • Performance Optimized: The software boasts efficient memory usage and fast processing.
The extension comes pre-configured to monitor the following critical security headers: 

Set-Cookie: High risk (Session cookies and security attributes)
Content-Security-Policy: High risk (CSP rules and restrictions)
X-Frame-Options: Medium risk (Clickjacking protection)
X-Content-Type-Options: Medium risk (MIME sniffing protection)
Referrer-Policy: Medium risk (Referrer information control)
Strict-Transport-Security: High risk (HTTPS enforcement)
X-XSS-Protection: Medium risk (XSS filtering settings)
Access-Control-Allow-Origin: Medium risk (CORS origin permissions)
Server: Low risk (Web server identification)
X-Powered-By: Low risk (Technology stack disclosure)

Usage

The extension automatically starts monitoring HTTP responses immediately after installation. To see the detected changes, visit the Header Change Notifier tab, and review alerts for high-risk changes in Burps main issue tracker. To configure settings, click on the Settings tab within the extension. Users can check or uncheck headers they want to monitor and add custom headers using the input field. Click on Save Settings to apply these changes. To export results, select the Export CSV button in the Header Changes tab, choose your desired save location, and save. The CSV exported contains all detected changes along with timestamps and risk levels. This extension can find application in different scenarios such as security testing, bug bounty hunting, development testing, compliance monitoring, and red team operations.
