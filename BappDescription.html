Header Change Notifier monitors HTTP response headers and alerts when they change between requests to the same URL. It helps identify security misconfigurations, server changes, and potential vulnerabilities by tracking critical security headers during testing.

Features

  • Passive scanner integration that respects scope settings and creates alerts in the issue tracker
  • Pre-configured tracking of critical security headers including CSP, HSTS, and cookie attributes
  • Risk level assessment that categorizes changes as Critical, High, Medium, or Low severity
  • Customizable header tracking with support for adding custom headers to monitor
  • CSV export functionality for reporting detected changes with timestamps
  • Clean table-based interface showing URL, header name, old and new values, and risk level

The extension monitors these security headers by default:

  • Set-Cookie (High) - Session cookies and security attributes
  • Content-Security-Policy (High) - CSP rules and restrictions
  • X-Frame-Options (Medium) - Clickjacking protection
  • X-Content-Type-Options (Medium) - MIME sniffing protection
  • Referrer-Policy (Medium) - Referrer information control
  • Strict-Transport-Security (High) - HTTPS enforcement
  • X-XSS-Protection (Medium) - XSS filtering settings
  • Access-Control-Allow-Origin (Medium) - CORS origin permissions
  • Server (Low) - Web server identification
  • X-Powered-By (Low) - Technology stack disclosure

Usage

  1. Once installed, the extension automatically monitors HTTP responses using passive scanning
  2. Configure your target scope in Burp Suite to control which requests are monitored
  3. Navigate the Header Change Notifier tab to view all detected changes in the table
  4. Review high-risk changes in the Scanner Issues tab where alerts are automatically created
  5. Customize tracked headers in the Settings tab by checking or unchecking headers, or adding custom headers
  6. Export detected changes to CSV by clicking the Export CSV button for reporting purposes
